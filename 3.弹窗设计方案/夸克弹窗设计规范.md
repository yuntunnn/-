# 夸克推广弹窗 — 设计规范与交互规范

> 本文档为方案A（多图轮播型）和方案B（活动营销型）的完整设计与交互规范，适用于**电脑桌面端**弹窗场景。

## 二、方案A — 多图轮播型 设计规范

### 2.1 轮播图区域

| 属性 | 规格 | 说明 |
|------|------|------|
| 区域高度 | **220px** | 轮播图固定高度 |
| 区域宽度 | 与弹窗等宽 **340px** | 100%填充 |
| 图片填充 | `object-fit: cover` | 裁切填充，保证无留白 |
| 圆角 | 仅顶部圆角（继承弹窗20px） | 底部直角衔接内容区 |

### 2.2 轮播图片资源规格

| 属性 | 规格 | 说明 |
|------|------|------|
| 图片尺寸 | **680×440px**（@2x） | 2倍图以适配高分屏 |
| 实际渲染 | 340×220px | CSS尺寸 |
| 图片格式 | **WebP** 优先，PNG 备用 | WebP体积更小，节省带宽 |
| 单张大小 | ≤ **80KB**（WebP）/ ≤150KB（PNG） | 保证加载速度 |
| 加载策略 | 首帧 `loading="eager"`，其余 `loading="lazy"` | 首帧立即加载，后续懒加载 |
| 图片数量 | **3帧** | AI搜索 / PDF编辑 / 无广告浏览 |

### 2.3 图片上的文案叠加技术方案

> **核心问题**：如何在图片上清晰展示白色文案？

#### 实现方式 — 底部渐变遮罩叠加

在每张轮播图上叠加一层 **从透明到半透明黑色的渐变遮罩**，然后在遮罩上放置白色文案。这样既保留了图片的视觉表现力，又确保文案清晰可读。

```css
/* 渐变遮罩层 */
.slide-overlay {
    position: absolute;
    inset: 0;
    background: linear-gradient(
        to bottom,
        rgba(0, 0, 0, 0.05) 0%,    /* 顶部：几乎透明，不遮挡图片主体 */
        rgba(0, 0, 0, 0.15) 40%,   /* 中部：轻微暗化 */
        rgba(0, 0, 0, 0.55) 100%   /* 底部：较明显暗化，确保文案可读 */
    );
}
```

| 叠加层属性 | 值 | 说明 |
|---------|------|------|
| 遮罩类型 | 从上到下的线性渐变 | 顶部透明 → 底部半透明黑 |
| 顶部透明度 | 5% 黑 | 不影响图片视觉 |
| 底部透明度 | 55% 黑 | 足够衬托白色文字 |
| 文案位置 | 遮罩底部，`padding-bottom: 18px` | 靠底对齐 |

#### 图内文案样式

| 元素 | 字号 | 字重 | 颜色 | 附加效果 |
|------|------|------|------|---------|
| 主标题 | **20px** | 700 Bold | `#FFFFFF` | `text-shadow: 0 1px 4px rgba(0,0,0,0.4)` |
| 副标题 | **13px** | 400 | `rgba(255,255,255,0.88)` | `text-shadow: 0 1px 3px rgba(0,0,0,0.3)` |

> **text-shadow 的作用**：即使图片背景较亮，文字阴影也能确保可读性。

## 四、桌面端适配建议

| 场景 | 建议 |
|------|------|
| 弹窗宽度 | 固定 340px，**不随屏幕缩放** |
| 高分屏 | 图片资源提供 @2x（680×440px） |
| 小屏笔记本（≤1366px） | 弹窗保持居中不变，无需调整 |
| 超宽屏（≥2560px） | 弹窗居中，遮罩覆盖全屏 |

## 五、性能优化建议

| 优化项 | 建议 |
|--------|------|
| 图片格式 | 优先使用 WebP，兜底 PNG |
| 图片压缩 | 单张 ≤ 80KB（WebP），质量 80-85% |
| 首帧加载 | `loading="eager"`，其余 `loading="lazy"` |
| CSS动画 | 仅使用 `transform` / `opacity`，触发 GPU 加速 |
| 倒计时 | `setInterval(1000)` 即可，避免 `requestAnimationFrame` 过度渲染 |
| 弹窗卸载 | 关闭后清除所有 `setInterval`，避免内存泄漏 |
